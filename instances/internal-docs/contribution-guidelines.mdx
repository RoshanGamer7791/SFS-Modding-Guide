# Contribution Guidelines

Thank you for your interest in contributing to the SFS Modding Guide! This document outlines the guidelines and best practices for contributing content, reporting issues, and submitting pull requests to ensure a smooth collaboration process.

There are several ways you can contribute to the SFS Modding Guide:

- **Content Contributions**: If you have expertise in SFS modding or related topics, consider writing new articles or improving existing ones. Please follow the [Style Guide](style-guide) to maintain consistency across the documentation.
- **Reporting Issues**: If you find any errors, outdated information, or have suggestions for improvements, please open an issue on the <a href={`${siteconfig.projectRepo}/issues`}>GitHub repository</a>.

## How to contribute

1. **Fork the Repository**: Start by forking the SFS Modding Guide repository on GitHub to your own account.
2. **Commit Changes**: Make your changes in a new branch. Ensure that your commits are clear and descriptive.
3. **Submit a Pull Request**: Once your changes are ready, submit a pull request to the main repository. Provide a clear description of the changes you made and any relevant context.
